# SAA-C03 問題: Amazon SQS を使用したアプリケーションコンポーネントの疎結合化

## 問題
ウェブアプリケーションが画像アップロードを受け付け、その後バックエンドで画像処理を行います。トラフィックが急増した際に、フロントエンドの応答性が低下し、画像処理に失敗することがあります。コンポーネントを疎結合化し、信頼性を向上させるための最適な構成はどれですか？

## 選択肢
A. フロントエンドからバックエンドの API を直接呼び出し、リトライ処理を実装する。
B. Amazon SQS キューを使用して、フロントエンドがメッセージを送信し、バックエンドがメッセージを処理する。
C. Amazon RDS のデータベースをキューとして代用し、ステータスを管理する。
D. AWS Lambda を使用して、画像を同期的に処理する。

## 正解
B

## 解説
Amazon SQS を使用することで、フロントエンドとバックエンドを疎結合にできます。フロントエンドはメッセージをキューに入れるだけで済み、バックエンドは自身のペースでメッセージを処理できます。トラフィック急増時もキューがバッファとして機能するため、システムの安定性が向上します。

*   **Aについて**: 同期的な呼び出しは疎結合ではなく、負荷が高い場合に依然としてフロントエンドに影響を与えます。
*   **Dについて**: 同期的な Lambda 呼び出しは、処理時間が長い場合にタイムアウトやエラーの原因となります。
*   **Cについて**: データベースをメッセージキューとして使用するのはアンチパターンであり、スケーラビリティやパフォーマンスの低下を招きます。
